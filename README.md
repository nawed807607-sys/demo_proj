# Zomato Bangalore Restaurant Delay Analysis

## 📌 Project Overview

This project analyzes the **Zomato Bangalore restaurant dataset** to identify restaurants that are more likely to experience delivery delays. Restaurants meeting the defined delay criteria are **red-flagged**, allowing users to quickly identify establishments that may have slower service.

The project demonstrates the complete data analysis workflow, including data cleaning, exploratory data analysis (EDA), feature engineering, and visualization using Python.

## 🎯 Objectives

* Clean and preprocess the restaurant dataset.
* Explore restaurant ratings, pricing, and other relevant features.
* Identify restaurants with potential delivery delays based on predefined conditions.
* Red-flag restaurants that exceed the delay threshold.
* Present insights through informative visualizations.

## 🛠️ Technologies Used

* Python
* Google Colab
* Pandas
* NumPy
* Matplotlib
* Seaborn

## 📊 Project Workflow

1. Import the dataset.
2. Perform data cleaning and preprocessing.
3. Conduct exploratory data analysis (EDA).
4. Apply delay detection logic.
5. Red-flag restaurants with delayed deliveries.
6. Visualize the results and summarize key insights.

## 📈 Key Insights

* Identified restaurants with a higher likelihood of delivery delays.
* Highlighted patterns between restaurant characteristics and delayed deliveries.
* Demonstrated how data analytics can support better customer decision-making.
